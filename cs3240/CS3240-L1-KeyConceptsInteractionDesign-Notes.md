## 1. Design Examples (Good vs. Not-So-Good)

### Good Design Examples

| Example | Why It's Good |
|---|---|
| **Google Search homepage** | **Minimalist design** unchanged for decades. Large search box + white space lets users **focus** on searching **without distractions** (no news, no widgets). Demonstrates the principle: **"less is more."** |
| **Apple product comparison** | Side-by-side comparison with essential specs. **Mimics real-life comparison behaviour**, making buying **decisions easier**. |
| **Spotify duplicate song warning** | Stops users from adding the same song to a playlist twice. **Anticipates likely and possible user errors** while still giving the user a choice ("Add anyway" / "Don't add"). |
| **"Return to top" button** (e.g., Dribbble) | Lets users **quickly** return to the top after long scrolling sessions. A small convenience that greatly improves navigation. |

### Not-So-Good Design Examples

| Example | Why It's Problematic |
|---|---|
| **Reddit mobile browser popup** | A **blocking popup** prompts users to download the app. The primary red button redirects to the App Store; the "Continue in browser" option is a **less obvious grey button** — prioritises business goals over user goals. |
| **Netflix autoplay previews** | Previews auto-play on hover, causing annoyance. **Pushes unwanted content.** A disable setting exists, but users must **know about it** and **remember to opt out** (opt-out vs. opt-in problem). |

### Discussion-Worthy Examples (Analyse These Yourself)

- **Shopee homepage** — heavy visual density, banners, promotions
- **NYU website** — grid-based tile layout, information density
- **Zara website** — extreme minimalism in fashion e-commerce

## 2. Core Definitions: Interaction, Interface, Experience

These three concepts form the foundation of the entire module:

| Concept | Definition |
|---|---|
| **Interaction** | A **reciprocal action or influence** — the back-and-forth between a user and a system. |
| **Interface** | **The means by which an interaction happens** — the touchpoint where user meets system. |
| **Experience** | **The way an interaction is perceived or felt** — the emotional and cognitive outcome. |

> **Key relationship:** Users perform actions through an *interface*, which facilitates an *interaction*, which produces an *experience*.

## 3. Three Pillars: IxD, UI, UX

| Discipline | Full Name | Core Focus |
|---|---|---|
| **IxD** | Interaction Design | Designing a **usable dialogue** between human and machine |
| **UI** | User Interface Design | Designing the interface's **look & feel** (visual design) and how information is **organized** (information architecture) |
| **UX** | User Experience Design | Designing for users' **experience** — emotions and **behaviour change** |

## 4. Interaction Design (IxD) — Deep Dive

IxD = Designing a **usable dialogue** between human and machine. It has two sub-concerns:

### 4a. Designing for Usability

Ensuring the interaction is:
- **Effective** — users can achieve their goals
- **Easy to learn** — low learning curve for new users
- **Efficient to use** — minimal effort for experienced users
- **Satisfying** — users feel good about the process

> Goal: Users achieve their objectives **without unnecessary effort or frustration**.

### 4b. Designing for Communication (Dialogue/Interaction)

The conversation between user and system:
- **User inputs:** clicks, taps, voice commands, gestures
- **System responses:** feedback, next steps, results

### Key Principle for Good IxD

> A **good interaction** feels **natural** and **predictable** if it:
> 1. Fits the **user's mental model** (how they expect things to work)
> 2. **Avoids overloading cognitive capacity** (keeping things simple and clear)

**Mental model** = the user's internal expectation of how a system should behave based on prior experience.

## 5. User Interface Design (UI) — Deep Dive

UI = Designing the interface's **visual design** + **information architecture**. It has two sub-concerns:

### 5a. Designing for Visual Appeal

How the interface *looks*:
- **Colors, typography, images, spacing, and layout style**
- Creates an **appealing, consistent, and brand-appropriate** appearance
- Goal: Make the product **aesthetically pleasing** while **supporting usability**

### 5b. Designing for Information (Information Architecture)

How information is *structured and organized*:
- **Content grouping** — related items together
- **Navigation scheme** — how users move between sections
- **Labeling** — clear names for sections and actions
- **Hierarchy** — visual and structural priority of content

> Goal: Users can **easily find what they need**; the interface is **logical and intuitive**.

---

## 6. User Experience Design (UX) — Deep Dive

UX = Designing for users' **emotions** and **behaviour change**. It has two sub-concerns:

### 6a. Designing for Emotions

- Creating products that **evoke emotions** and **influence user actions**
- Crafting experiences that trigger feelings: **joy, trust, excitement, urgency**
- Goal: **Deepen engagement** and make interactions **memorable**

### 6b. Designing for Behavioral Change

- **Guiding users toward desired actions or habits**
- Making users feel **motivated, rewarded, or supported**
- Should feel **natural and positive** (not manipulative)

> Example: A fitness app that celebrates streaks (joy + motivation) to encourage daily exercise (behavioral change).

| Type | Description | Examples |
|---|---|---|
| **Conversational** | Dialogue-based | Chatbots, voice assistants |
| **Manipulation** | Directly changing on-screen/physical objects | Drag-and-drop, pinch-to-zoom |
| **Instructional** | Step-by-step commands | Setup wizards, command line |
| **Exploration** | Open-ended or immersive | VR tours, Google Earth, open-world games |

| Type | Description |
|---|---|
| **Graphical User Interface (GUI)** | Windows, icons, menus |
| **Web Interface** | Browser-based; responsive layouts, hyperlinks, forms |
| **Mobile Interface** | Smartphones/tablets; touch gestures, limited screen real estate |
| **Voice User Interface (VUI)** | Voice commands, speech output |
| **Gesture-based Interface** | Kinect, Leap Motion, touchscreen gestures |
| **Tangible User Interface** | Physical objects as input devices |
| **Brain-Computer Interface** | Neural signals controlling systems |

### Experience Types

| Type | Description | Examples |
|---|---|---|
| **Productive** | Helping users achieve tasks efficiently | Productivity apps |
| **Sensory / Aesthetic** | Pleasing the senses (visual, audio, haptic) | Beautiful portfolios |

| **Immersive** | Making the user feel "inside" the environment | VR, AR games |
| **Social** | Connecting people | Multiplayer games, social platforms |
| **Collaborative** | Enabling people to work together toward a shared goal | Google Docs, Figma, Miro |

## 8. Product Design vs. Service Design

### Product Design

- Focuses on **what the user directly interacts with**: screens, buttons, flows, interactions
- Designing a booking task, checkout flow, or dashboard = product design

> **Food Delivery Example:** The screens where you browse food and place an order.

### Service Design

- Looks at the **experience over time** — the full end-to-end journey
- Asks: What happens **before** the user opens the app? What happens **after**? What happens **when things go wrong**? What **systems, people, and rules** are involved?

> **Food Delivery Example:** Order confirmation, delivery tracking, refunds, customer support, and what happens if something goes wrong.

### Key Distinction

| | Product Design | Service Design |
|---|---|---|
| **Scope** | The product itself (screens, UI) | The entire service ecosystem over time |
| **Focus** | Direct user interaction | Before, during, after + edge cases |
| **Involves** | Designers, developers | Designers + operations, support, logistics |

## 9. Affordance and Signifier

### Affordance

> **Affordances** are **points of interaction between object and perceiver** that allow the perceiver to **understand the object's features — what actions are possible**.

- A door handle *affords* pulling; a flat plate *affords* pushing
- A button on screen *affords* clicking
- Affordances exist whether or not the user perceives them

### Signifier

> **Signifiers** are designed **to communicate an object's affordances** — they tell the user *what to do*.

- A "PUSH" / "PULL" sign on a door = **signifier** (explains the affordance because the handle design alone is confusing)
- A **greyed-out button** = signifier indicating the button is **inactive/disabled**
- Placeholder text in a form field ("email@email.com") = signifier showing what to type

### Affordance vs. Signifier — Comparison

| | Affordance | Signifier |
|---|---|---|
| **What it is** | The actual action possibilities of an object | A cue that communicates the affordance |
| **Nature** | Can be physical or perceived | Always a perceptible signal (visual, auditory, etc.) |
| **Example (physical)** | A handle you can grasp | A "PULL" sign next to the handle |
| **Example (digital)** | A button that can be clicked | The button's color, shadow, or label that says "Submit" |

> **Exam tip:** Be able to identify affordances and signifiers in any given design example, and explain whether the signifiers correctly communicate the affordances.

## 10. The New Age: AI in Interaction Design

### AI-Powered Interaction Patterns

Interactions are getting **faster, more personal, and more predictive**:

| Pattern | Examples |
|---|---|
| **Predictive Text & Autocomplete** | Gmail Smart Compose, chat app suggestions |
| **Personalized Recommendations** | Netflix, Spotify, Amazon product suggestions |
| **Smart Search** | Google Lens image search, in-app natural language queries |
| **Voice Assistants** | Siri, Alexa, in-app voice commands |
| **Image/Video Tagging** | Automatic categorization in Google Photos |
| **AI-powered Accessibility** | Real-time captions, screen readers with context awareness |

### How AI Changes IxD, UI, and UX

| Dimension | Impact of AI |
|---|---|
| **Interaction** | From **manual to assisted** (AI suggests next steps) |
| | From **reactive to proactive** (system acts before user asks) |
| | From **static to adaptive** (interface changes based on behaviour) |
| **UI** | AI can alter the **look & feel dynamically** (adaptive layouts, personalized dashboards) |
| **UX** | AI changes **flows and behaviours** (anticipating needs, removing steps) |

> **Example:** In a travel app, AI might:
> - Pre-fill flight details based on past bookings
> - Recommend hotels near your destination
> - Automatically suggest a packing list

### Real-World AI Integration Examples

- **Google Lens** — image-based search: take action on text, learn about the world, identify plants/animals, find similar products
- **Khanmigo (Khan Academy)** — AI-powered tutor for brainstorming activities, encouraging learning, and running quizzes/games/debates

### Open Questions for AI in Interaction Design

These are important to think about for exams:

1. **When should we use AI** in our products?
2. **How do we make AI-powered products easy to use** for everyone?
3. **How do users understand** what the AI is doing? (transparency/explainability)
4. **What happens when AI gets it wrong?** (error handling, trust recovery)

## 11. Summary and Key Takeaways

### What Makes a Good Design

1. Should let users **achieve their goals effectively**
2. Should assist users in taking **quick decisions**
3. Should be **aesthetically pleasing** with **well-organised information**

### For a Usable Interaction, Users Must Be Able To:

1. **Figure out what actions to take** to achieve a specific goal
2. **Understand the results** of those actions

### Concept Map (How Everything Connects)

```
Interaction Design (IxD)
    |-- Designing for Usability (effective, learnable, efficient, satisfying)
    |-- Designing for Communication (user inputs <-> system responses)
    |-- Key: fits mental model, avoids cognitive overload

User Interface Design (UI)
    |-- Designing for Visual Appeal (colors, typography, layout)
    |-- Designing for Information Architecture (grouping, navigation, labeling, hierarchy)

User Experience Design (UX)
    |-- Designing for Emotions (joy, trust, excitement, urgency)
    |-- Designing for Behavioral Change (motivation, rewards, habits)

Product Design = the screens/flows the user directly interacts with
Service Design = the full experience over time (before, during, after, edge cases)

Affordance = what actions are possible
Signifier = what communicates those possible actions to the user

AI is shifting design from manual->assisted, reactive->proactive, static->adaptive
```
